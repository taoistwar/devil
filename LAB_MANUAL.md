## 实验准备

### 环境要求

```bash
# 克隆项目
git clone https://github.com/taoistwar/devil.git
cd devil

# 构建
cargo build --release

# 配置 API 密钥
export DEVIL_API_KEY="sk-ant-xxx"
```

### 实验结构

每个实验包含：
- **目标**：学习什么
- **前置条件**：需要什么
- **步骤**：操作指南
- **预期结果**：应该看到什么
- **思考题**：加深理解

### 启动方式

```bash
# 单次任务模式
devil run "你的任务"

# 交互式模式
devil repl
```

## 实验 1：你好，世界

### 目标

了解 MonkeyCode Agent 的基本交互方式。

### 前置条件

- ✅ 安装完成
- ✅ `DEVIL_API_KEY` 已设置

### 步骤

#### 任务 1.1：首次对话

```bash
devil run "你好，请介绍一下你自己"
```

**预期结果**：Agent 应该回复并介绍自己。

#### 任务 1.2：对话式交互

```bash
devil repl

# 在 REPL 中输入
> 你好

> 你能做什么？

> 用中文回答
```

**预期结果**：Agent 用中文回复，展示对话能力。

#### 任务 1.3：多轮对话

```
> 帮我创建一个简单的 Rust 函数，计算斐波那契数列

> 能解释一下这段代码吗？

> 优化一下性能
```

**预期结果**：Agent 保持上下文，理解多轮对话。

### 思考题

1. Agent 如何保持对话上下文？
2. 对话轮次过多可能会遇到什么问题？

## 实验 2：文件操作

### 目标

学习使用 Agent 进行文件读取、编辑和管理。

### 前置条件

- ✅ 完成实验 1
- ✅ 项目中有 Rust 源码

### 步骤

#### 任务 2.1：读取文件

```bash
devil run "读取 Cargo.toml 的内容，并总结这个项目的依赖"
```

**预期结果**：Agent 显示文件内容并提供摘要。

#### 任务 2.2：分析代码结构

```bash
devil run "分析 src/ 目录下的 Rust 文件，列出所有模块"
```

**预期结果**：Agent 扫描目录并列出模块结构。

#### 任务 2.3：编辑文件

```bash
devil run "
1. 在 /tmp 目录下创建 hello.rs 文件
2. 文件内容是一个 Hello World 程序
3. 使用 rustc 编译并运行
"
```

**预期结果**：
- 创建 `hello.rs`
- 编译成功
- 运行输出 "Hello, World!"

#### 任务 2.4：批量编辑

```bash
devil run "
1. 在 /tmp/test_rust/ 目录创建 3 个文件：main.rs, lib.rs, mod.rs
2. main.rs 调用 lib.rs 的函数
3. 编译运行验证
"
```

**预期结果**：Agent 创建完整的小型 Rust 项目结构。

### 思考题

1. Agent 如何决定使用 Read 还是 Glob 工具？
2. 编辑敏感文件时有什么保护机制？

## 实验 4：记忆系统

### 目标

了解如何使用记忆系统保存和检索信息。

### 前置条件

- ✅ 完成实验 1

### 步骤

#### 任务 4.1：查看记忆

```bash
devil run "/memory list"
```

**预期结果**：显示当前所有记忆条目。

#### 任务 4.2：添加用户记忆

```bash
devil run "/memory add user 我的名字 我的名字是 Alice"
```

**预期结果**：记忆已保存的确认消息。

#### 任务 4.3：添加项目记忆

```bash
devil run "/memory add project 项目类型 这是一个 Rust 语言项目"
```

**预期结果**：项目记忆已保存。

#### 任务 4.4：检索记忆

```bash
devil run "/memory list user"
```

**预期结果**：只显示用户类型的记忆。

#### 任务 4.5：跨会话记忆

1. 添加记忆后退出：
```
> /exit
```

2. 重新启动：
```bash
devil repl
```

3. 查询记忆：
```
> /memory list
> 记得我的名字是什么吗？
```

**预期结果**：Agent 能回忆起之前保存的信息。

### 思考题

1. 记忆系统如何实现持久化？
2. 不同类型的记忆有什么区别？

## 实验 7：协调器模式

### 目标

学习使用协调器模式进行多 Agent 并行任务。

### 前置条件

- ✅ 完成实验 1-6
- ✅ 理解并行任务概念

### 步骤

#### 任务 7.1：启用协调器模式

```bash
devil repl

> /coordinator status
> /coordinator on
```

**预期结果**：
- 显示协调器状态
- 模式已启用

#### 任务 7.2：并行研究

```bash
devil run "
启用协调器模式后：
1. 派发一个 Worker 研究 src/agent 的结构
2. 派发另一个 Worker 研究 src/tools 的结构
3. 汇总两个 Worker 的发现
"
```

**预期结果**：
- 多个 Worker 并行执行
- 结果以 `<task-notification>` 格式返回
- 协调器综合结果

#### 任务 7.3：任务分配

```bash
devil run "
使用协调器模式：
1. 研究项目中有多少个 crate
2. 列出每个 crate 的主要模块
3. 生成项目结构报告
"
```

**预期结果**：协调器派发任务，汇总研究报告。

#### 任务 7.4：结果聚合

```bash
devil run "
1. 派发 3 个 Worker 并行分析不同目录
2. 等待所有 Worker 完成
3. 聚合所有发现生成总结
"
```

**预期结果**：
- 3 个并行 Worker
- 协调器等待所有结果
- 生成统一报告

### 思考题

1. 协调器模式和普通模式的区别是什么？
2. Worker 之间如何共享数据？

## 实验答案参考

### 实验 1

1. Agent 通过消息历史保持上下文
2. 上下文过长会触发压缩或达到限制

### 实验 2

1. Agent 根据任务选择合适的工具
2. 通过权限系统保护敏感文件

### 实验 3

1. Glob 找文件，Grep 搜索内容
2. 可以搜索特定注释模式

### 实验 4

1. 记忆存储在 `~/.claude/projects/<project>/memory/`
2. 不同类型记忆用途不同

### 实验 5

1. Todo 是内部任务管理
2. 任务状态在会话中保持

### 实验 6

1. 斜杠命令是自然语言的补充
2. 通过插件系统扩展

### 实验 7

1. 协调器派发独立 Worker
2. 通过 Scratchpad 或协调者综合

### 实验 8

1. MCP 提供标准化工具集成
2. 通过 JSON-RPC 通信
