### 依赖关系详解

| 上游规范 | 下游规范 | 依赖类型 | 说明 |
|---------|---------|---------|------|
| 001 | 002 | 强依赖 | CLI 是 Agent 的入口方式 |
| 001 | 003 | 强依赖 | 工具是 Agent 执行任务的核心能力 |
| 001 | 006 | 强依赖 | 上下文是 Agent 理解对话的基础 |
| 002 | 005 | 强依赖 | 斜杠命令通过 CLI 调度 |
| 003 | 004 | 强依赖 | 所有工具必须通过权限系统 |
| 003 | 006 | 弱依赖 | 上下文注入工具结果 |
| 004 | 008 | 中依赖 | Coordinator 需要权限控制 |
| 005 | 008 | 中依赖 | /coordinator 命令启用协调模式 |
| 006 | 007 | 弱依赖 | Memory 影响上下文内容 |
| 007 | 008 | 弱依赖 | 协调者可能需要查询记忆 |
| 008 | 009 | 弱依赖 | Web 服务器暴露 Agent 能力 |

## 3. 统一接口规范

### 3.1 错误处理规范

所有模块必须遵循以下错误处理规范：

```rust
// 错误层次结构
pub enum DevilError {
    /// 用户输入错误（4xx）
    UserInput {
        code: ErrorCode,
        message: String,
        field: Option<String>,
    },
    /// 权限错误（403）
    Permission {
        code: ErrorCode,
        message: String,
        required_permission: String,
    },
    /// 资源错误（409/507）
    Resource {
        code: ErrorCode,
        message: String,
        resource_type: String,
    },
    /// 外部服务错误（502/503）
    ExternalService {
        code: ErrorCode,
        message: String,
        service: String,
    },
    /// 内部错误（500）
    Internal {
        code: ErrorCode,
        message: String,
        stack_trace: Option<String>,
    },
}

#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash)]
pub enum ErrorCode {
    // 用户输入错误 (E1xxx)
    InvalidInput = 1001,
    MissingRequiredField = 1002,
    InvalidFormat = 1003,
    
    // 权限错误 (E2xxx)
    PermissionDenied = 2001,
    InsufficientPermissions = 2002,
    OperationNotAllowed = 2003,
    
    // 资源错误 (E3xxx)
    ResourceNotFound = 3001,
    ResourceConflict = 3002,
    ResourceExhausted = 3003,
    
    // 外部服务错误 (E4xxx)
    LLMProviderError = 4001,
    MCPConnectionError = 4002,
    NetworkError = 4003,
    
    // 内部错误 (E5xxx)
    InternalError = 5001,
    NotImplemented = 5002,
    UnexpectedState = 5003,
}
```

### 3.2 日志规范

```rust
// 日志级别使用场景
log::error!  - 系统级故障，需要人工干预
log::warn!   - 可恢复的异常，可能导致功能降级
log::info!   - 重要业务事件（Agent 启动/停止、任务完成）

log::trace!  - 详细追踪信息（消息流转、状态变化）

// 日志格式
// time | level | module | message | context(k=v)
```

## 4. 模块边界定义

| 模块 | 职责 | 清晰的边界 |
|------|------|----------|
| `cli` | 命令行解析和调度 | 只负责解析和分发，不执行业务逻辑 |
| `gateway` | HTTP/WebSocket 接入 | 只负责协议转换，不处理业务 |
| `agent` | Agent 核心逻辑 | 状态管理、循环控制 |
| `tools` | 工具执行 | 只负责工具的定义和执行，不涉及权限 |
| `permissions` | 权限判断 | 独立于工具的权限检查服务 |
| `context` | 上下文管理 | 消息压缩、Token 预算 |
| `memory` | 记忆存储 | 持久化存储，与 Agent 解耦 |
| `providers` | 外部服务 | LLM API 调用封装 |

## 6. 会话生命周期

```
Session 生命周期
│
├─ 创建 (Session::new)
│   ├─ 加载配置
│   ├─ 初始化工具注册表
│   ├─ 加载记忆 (spec-007)
│   └─ 注入初始上下文 (spec-006)
│
├─ 运行 (session.run)
│   ├─ 解析用户输入
│   ├─ 权限检查 (spec-004)
│   ├─ 执行工具 (spec-003)
│   ├─ 更新上下文 (spec-006)
│   ├─ 调用 LLM (Coordinator 模式下派发 Worker, spec-008)
│   └─ 返回响应
│
├─ 保存 (session.save)
│   ├─ 持久化记忆 (spec-007)
│   ├─ 保存上下文状态
│   └─ 更新会话索引
│
└─ 清理 (session.close)
    ├─ 关闭工具连接
    ├─ 刷新缓冲区
    └─ 释放资源
```

## 7. 版本兼容性策略

### API 版本

| 版本 | 状态 | 说明 |
|------|------|------|
| v1 | 当前 | 基础功能 |
| v1.1 | 规划 | 批量工具调用 |
| v2 | 规划 | 流式响应改进 |

### 向后兼容性

- 工具参数：新增可选参数不破坏现有调用
- 错误码：只新增，不修改已有错误码含义
- 配置项：新增配置项有默认值，不影响现有配置
